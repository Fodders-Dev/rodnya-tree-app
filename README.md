# Родня — семейное дерево и семейная социальная сеть

![image](https://github.com/user-attachments/assets/ac73f72b-8d5a-4115-949e-023c27b76639)

**Родня** — это Flutter-приложение для семейного дерева, общения между родственниками и хранения семейной истории. Ближайшая цель проекта — удобный ежедневный Android/web MVP с плотным интерфейсом, стабильным медиа-path и совместным ведением дерева.

Публичное и внутреннее активное имя продукта — `Родня` / `rodnya`. Старые технические следы постепенно вычищаются из репозитория, сборок и release tooling.

## Что уже является приоритетом релиза

- auth и onboarding
- семейное дерево: просмотр, CRUD, совместное редактирование
- 1:1 чат и уведомления
- посты, профиль, базовая загрузка медиа
- стабильные Android и web сборки
- семейные события и полезный домашний экран без лишней "текстовой массы"

## Ключевые пользовательские сценарии

- создание и ведение семейного дерева
- просмотр родственников, профилей, фото и памятных дат
- публикация постов внутри семейного круга
- личные чаты между зарегистрированными пользователями
- уведомления о важных событиях и активности
- совместное пополнение семейной информации несколькими участниками

## Ближайшие продуктовые волны

- выравнивание бренда и технических идентификаторов под `Родню`
- server-driven likes и выравнивание production parity
- полноценные `24h stories`
- расширенный семейный календарь: дни рождения, годовщины, памятные даты, праздники РФ и православные праздники
- история изменений дерева и галерея фото родственников
- упрощение Home/Tree/Profile/Relatives до action-first UX

## Текущий технический вектор

- Flutter
- custom API + PostgreSQL
- object storage для медиа
- provider-driven state + thin screens
- поэтапный уход от Firebase-hosted и hosted Supabase зависимостей, если они мешают доступности в России
- Android и web как основные релизные платформы; iOS позже

Подробный execution plan зафиксирован в [docs/rodnya_release_plan_2026-04-13.md](docs/rodnya_release_plan_2026-04-13.md).

## Начало работы

1. Клонировать репозиторий.
2. Установить зависимости:

```bash
flutter pub get
```

3. Для реальной web-проверки собирать и локально сервировать release bundle:

```bash
flutter build web
python -m http.server 3000 --bind 127.0.0.1
```

4. Для compile-only проверки можно использовать:

```bash
flutter build web --no-wasm-dry-run
```
