# Перенос продакшена в РФ — план и выбор хостинга

> 2026-06-12. Текущее: бэк + Postgres на Hostkey NL (212.69.84.167). Медиа-S3 — регион `ru-msk` (уже РФ ✓). UniSender — РФ ✓. LiveKit — [проверить, где крутится: смотри RODNYA_LIVEKIT_URL в env сервера — если на том же NL-хосте, едет вместе с бэком].

## Что переносим
1. Node-бэкенд (systemd/докер — как сейчас на Hostkey).
2. PostgreSQL (одна БД `rodnya_state` — JSONB-документ + таблицы).
3. env-файл (`/etc/rodnya-backend.env` — бэкап уже делали: `.bak-rustore-…`).
4. [LiveKit, если self-hosted на том же сервере.]
DNS: `api.rodnya-tree.ru` → новый IP. Web-фронт деплоится GitHub Actions — только секреты `SERVER_HOST`/`BACKEND_SERVER_*` обновить.

## Выбор хостинга (решение твоё, критерии мои)
| Критерий | Timeweb Cloud | Selectel | Yandex Cloud |
|---|---|---|---|
| Простота (панель, для одиночки) | ★★★ | ★★ | ★★ |
| VPS под наш профиль (4 vCPU/8GB) | есть | есть | есть |
| Управляемый Postgres (опция) | есть | есть | есть |
| S3-совместимое хранилище | есть | есть (наш ru-msk может уже быть тут — проверь, чей бакет!) | есть |
| Бюрократия/оплата физлицом | минимальная | средняя | средняя |
**Рекомендация:** если текущий S3 `ru-msk` — это VK Cloud или Selectel, бери того же провайдера под VPS (меньше сущностей). Иначе — Timeweb как самый простой. Цены смотри на сайтах — диапазон для нашего профиля ~1.5-3 тыс ₽/мес за VPS.

## План переноса (минимальный простой, ~1 вечер + репетиция)
1. **Подготовка (без простоя):** новый VPS → докер/Node той же версии → Postgres → прогнать деплой-скрипт на поддомен `api2.rodnya-tree.ru` → env скопировать (секреты руками) → восстановить свежий дамп БД → смоук `tool/prod_route_smoke.mjs --api https://api2...` (я прогоню).
2. **Репетиция:** сутки гоняем api2 параллельно (PW-смоук на него), сверяем логи.
3. **Переключение (окно ~15 мин, ночью):** включить maintenance-режим (бэк отвечает 503 с человеческим текстом) → финальный дамп → restore на новый → DNS `api.` → новый IP (TTL заранее снизить до 300с) → смоук → выключить maintenance.
4. **После:** обновить GitHub Secrets деплоя → тестовый deploy → PW зелёный → старый сервер morозим (не удаляем 2 недели) → впиши провайдера/город в Политику (п.4) и шпаргалку РКН.
5. **ОРИ-архив (следующий батч):** retention-хранилище на этом же VPS или отдельном бакете РФ — после параметров от юриста.

## Роли
- **Артём:** регистрация у провайдера, оплата, доступы мне (SSH-ключ деплоя), снижение TTL DNS заранее.
- **Я:** всё остальное по шагам 1-4 + смоуки + откат-план (DNS назад, старый сервер живой).
