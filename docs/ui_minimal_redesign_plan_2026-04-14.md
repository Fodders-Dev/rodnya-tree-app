# UI Minimal Redesign Plan

## Цель
Сделать `Родню` легче, спокойнее и интуитивнее: меньше постоянных текстов, меньше тяжёлых карточек, больше иконок, статусов и прямых действий.

## Визуальный тезис
Тихий `liquid glass` для семейного приложения: светлый фон, мягкие полупрозрачные панели, плотная типографика, один тёплый акцент, минимум рамок и минимум декоративного шума.

## Контент-план
1. Навигация: быстрый доступ, крупные иконки, короткие подписи.
2. Рабочая зона: дерево, лента, список родственников, профиль.
3. Контекст: только статусы, счётчики и действия.
4. Пустые состояния: одна строка смысла, одно действие.

## Interaction thesis
- Плавающие стеклянные панели вместо тяжёлых белых карточек.
- Иконка + короткий статус вместо длинных описательных абзацев.
- Мягкие hover/press состояния и компактные pills/chips для фильтров и быстрых действий.

## Правила редизайна
- Убирать explanatory text, если смысл уже понятен из иконки, заголовка и действия.
- Не держать постоянные большие banners, если это не ошибка, first-run или pending state.
- Любая вторичная информация должна быть в chips, counters, bottom sheets или overflow.
- Основной акцент: бирюзово-зелёный. Остальное нейтральное.
- Повседневные экраны не должны выглядеть как набор карточек.
- Пустые блоки должны быть компактными и не ломать ритм экрана.

## Приоритет по экранам

### Wave 1. Shared system
- Пересобрать `AppTheme` под стеклянные светлые поверхности.
- Облегчить `AppBar`, `FAB`, `Input`, `Buttons`, `Chips`, `Cards`.
- Заменить стандартный `BottomNavigationBar` на более лёгкую glass-nav.

### Wave 2. Home
- Упростить header до аватара, имени, 2-4 статусов и 2-3 действий.
- Сжать stories и events до компактных модулей.
- Сделать пустую ленту короче: иконка, одна строка, одно действие.
- Убрать тяжёлую рамочную структуру слева и в центре.

### Wave 3. Tree
- Максимально отдать экран canvas дерева.
- Пустые/ошибочные состояния сделать компактнее.
- Toolbar и quick actions перевести в pills и icon actions.
- Свести постоянные подписи к минимуму.

### Wave 4. Profile
- Убрать длинные текстовые секции.
- Оставить: аватар, имя, короткие stats, stories, posts, linked person.
- Notes и вторичный контент сделать менее тяжёлыми визуально.

### Wave 5. Relatives
- Сократить верхний banner и боковую сводку.
- Упростить карточки людей: фото, имя, 1 строка роли, status chips, quick actions.
- Empty/no-tree states сделать короче и визуально спокойнее.

## Компоненты для зачистки
- `EmptyStateWidget`
- `EventCard`
- `StoryRail`
- `PostCard`
- `InteractiveFamilyTree` inspector
- крупные summary/side panels

## Что проверять после каждой волны
- Нет ли длинных абзацев на `Home`, `Tree`, `Profile`, `Relatives`.
- Понимается ли экран по заголовкам, иконкам и основным действиям.
- Не перегружен ли экран рамками, тенями и зелёными заливками.
- Сохранились ли `stories`, `likes`, `tree history`, `relative gallery`.

## Acceptance
- На desktop и mobile интерфейс ощущается легче и чище.
- Основные действия читаются за 2-3 секунды без помощи длинного текста.
- Визуальный язык единый на `Home`, `Tree`, `Profile`, `Relatives`.
- Browser smoke и релевантные widget tests проходят.
